# UI Theme Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Recreate the warm-paper/dotted-grid visual theme from `design_handoff_ui_theme/` across the Options page (sidebar shell + all 7 sections) and the Side Panel (all states), using Tailwind utility classes on the existing React components — presentation-layer only.

**Architecture:** Tailwind's theme is extended with the design's exact color/font/radius tokens (`tailwind.config.ts`). Two font packages are bundled via `@fontsource` (no runtime Google Fonts fetch, required for a Chrome extension). Every component keeps its existing element types, `id`/`htmlFor` pairs, `aria-label`s, and — critically — every exact text string that an existing test locks in via `getByRole(..., { name: '...' })` or `getByText('...')`; only `className` values and, where the design calls for a materially different structure (e.g., toggle switches, status pills, the sidebar nav), the surrounding JSX markup change. No `chrome.storage`/messaging/repository code is touched by this plan.

**Tech Stack:** Same as the existing scaffold — TypeScript + React 19 + Tailwind CSS + Vite (CRXJS). New: `@fontsource/instrument-sans`, `@fontsource/jetbrains-mono`.

## Global Constraints

- **This is a presentation-only plan.** Do not modify any repository, storage, messaging, schema `.transform` logic, or content-script code. Do not add new `chrome.storage` reads/writes, new message types, or new validation rules.
- **TDD adaptation for a presentation-only plan:** there is no new business behavior to drive with a failing test. Each task's test gate is instead: confirm the existing test(s) for that file pass *before* your change (baseline), apply the styling/structure change, confirm the *same* tests still pass *after* (regression-proof), unless a task explicitly says to update a specific test assertion (only two tasks do this, and only for the literal reasons stated in that task — never silently).
- **Every existing locked test string is preserved verbatim**, per an explicit decision already made with the user: button text, link text, status text, and error/success messages keep their EXACT current copy — even where `design_handoff_ui_theme/README.md`'s mockup shows different wording (e.g., mockup says "Save changes", the app keeps "Save"; mockup says "Export JSON", the app keeps "Export data"; mockup says "Autofill this page", the app keeps "Autofill current page"; mockup says "Add experience" is on both a "+" top button and a "Save entry" submit button, the app's existing submit button keeps its exact toggling text "Add experience"/"Update experience" and only the new top button gets the mockup's "+ Add experience" wording, since that's a genuinely new element with no prior locked name). Apply the design's *visual* treatment (color, spacing, radius, pill/card shape) to the *existing* copy — do not silently adopt the mockup's alternate copy anywhere in this plan.
- **One explicit exception, already agreed with the user:** the Side Panel's autofill-result display switches from a single locked sentence to the design's 2×2 stat-card grid, and `src/sidepanel/App.test.tsx`'s one test that asserted the old combined sentence is REPLACED with an assertion checking the four individual numbers. This is the only test file this plan is authorized to change, and only for that one test case (Task 10 spells out the exact new test).
- **Design tokens** (verbatim from `design_handoff_ui_theme/README.md`): colors `canvas #F6F4F1`, `surface #FFFFFF`, `raised #FDFCFB`, `ink #1C1A17`, `body #45413B`, `muted #6E6961`, `faint #9B958C`, `line #EAE6E0`, `line-strong #DDD8D0`, `hairline #F1EEE9`, `primary #6C55C8`, `teal #2AA79B`, `amber #E4B33C`, `success #1E8E5A`, `warning #B98900`, `danger #C4453C`. Fonts: `sans` = Instrument Sans, `mono` = JetBrains Mono. Radii: `card 14px`, `panel 20px`, `input 10px`, `badge 6px`.
- **One-off colors used by specific components but not promoted to the token list** (per the design HTML, used via Tailwind arbitrary-value syntax `bg-[#XXXXXX]` at the point of use, not added to `tailwind.config.ts`): `#E3DFD8` (raised-panel border), `#C9C3BA` (inactive nav dot / disabled text), `#EDE8F9`/`#E4F3F1`/`#FAF1DC` (avatar tints), `#FBFAF8` (row hover), `#FBF7EE`/`#EAD9B0` (sensitive row background/pill border), `#F0F7F4`/`#CFE5DB` (saved-toast background/border), `#2E2B26` (ink button hover).
- **Design System Reference** — reusable Tailwind class strings every task below applies consistently (do not invent alternate spacing/sizing for the same component type):
  - **Page panel** (page-level container, radius 20px): `bg-raised border border-[#E3DFD8] rounded-panel shadow-[0_12px_32px_rgba(28,26,23,0.08)]`
  - **Card** (radius 14px): `bg-surface border border-line rounded-card shadow-[0_1px_2px_rgba(28,26,23,0.04)]`
  - **Section label** (mono uppercase): `font-mono text-[11px] text-muted uppercase tracking-[0.08em]`
  - **Field label**: `text-[12px] font-semibold text-body`; optional-field variant: `text-[12px] font-semibold text-faint` with a trailing `<span className="font-normal"> · optional</span>`
  - **Text input**: `font-sans text-[13px] text-ink bg-surface border border-line-strong rounded-input px-3 py-2.5 outline-none w-full focus:border-primary transition-colors`
  - **Mono input** (URLs, keys): same as text input but `font-mono text-[12px]`
  - **Primary button (ink)**: `font-sans text-[13px] font-semibold bg-ink text-white rounded-input px-[18px] py-[10px] hover:bg-[#2E2B26] transition-colors duration-150 disabled:opacity-50`
  - **Secondary button**: `font-sans text-[13px] font-semibold bg-surface text-ink border border-line-strong rounded-input px-[18px] py-[10px] hover:bg-[#FBFAF8] transition-colors duration-150`
  - **Status pill** (wraps existing locked text, dot color varies): `inline-flex items-center gap-1.5 border border-line rounded-full px-2.5 py-[3px] text-[11px] font-medium bg-surface` + a leading `<span className="w-1.5 h-1.5 rounded-full bg-{color}" />`
  - **Toggle switch** (real `<input type="checkbox">`, visually hidden, styled via a sibling track): wrapping `<label className="relative inline-flex items-center cursor-pointer flex-shrink-0">`, input `className="sr-only peer"`, track `<span className="w-[34px] h-5 bg-line-strong peer-checked:bg-success rounded-full transition-colors duration-150 relative after:content-[''] after:absolute after:top-0.5 after:left-0.5 after:bg-white after:rounded-full after:h-4 after:w-4 after:shadow-[0_1px_2px_rgba(28,26,23,0.2)] after:transition-all after:duration-150 peer-checked:after:translate-x-[14px]" />`
  - **Segmented control**: track `inline-flex bg-hairline rounded-input p-[3px] gap-0.5`, active segment `bg-white rounded-[8px] shadow-[0_1px_2px_rgba(28,26,23,0.08)] text-[12px] font-semibold px-3.5 py-1.5 text-ink`, inactive segment `text-[12px] font-medium text-muted px-3.5 py-1.5`
  - **Focus ring** (a11y, applied once globally in Task 1, not per-component): `:focus-visible { outline: 2px solid #6C55C8; outline-offset: 2px; }`
- **Manual verification requirement:** per the frontend-change guideline, Tasks 2 through 10 each end with starting the Vite dev server and visually comparing the rendered page against the corresponding screen id in `Extension Theme.dc.html` in a browser before being considered done — automated tests only prove no *regression*, not visual correctness.

function Wordmark({ size = 18 }: { size?: number }) {
  const dot = Math.round(size * 0.61)
  return (
    <div className="flex items-center gap-1.5">
      <span className="font-sans font-bold tracking-[-0.02em]" style={{ fontSize: size }}>

        />
        <span
          className="rounded-full bg-amber"

        />
      </span>
    </div>
  )
}

export function App() {
  const [activeTab, setActiveTab] = useState<TabKey>('personal')

  return (

                  />

              )

          </div>
        ))}
        <div className="mt-auto border-t border-line pt-3.5 px-2.5 flex flex-col gap-0.5">
          <span className="text-[13px] font-semibold text-ink">Local-only storage</span>

        {activeTab === 'work' && <WorkExperiencePage />}
        {activeTab === 'education' && <EducationPage />}
        {activeTab === 'answers' && <AnswerBankPage />}
        {activeTab === 'applications' && <ApplicationRecordsPage />}
        {activeTab === 'import-export' && <ImportExportPage />}
        {activeTab === 'privacy' && <PrivacySettingsPage />}
      </main>
    </div>
  )
}

import { useState, type ChangeEvent } from 'react'
import { buildExportBundle, restoreFromBundle } from '../../shared/storage/export-import'
import type { ExportBundle } from '../../shared/types/export-bundle'
import { exportBundleSchema } from '../../shared/types/export-bundle-schema'

type ImportStatus = 'idle' | 'success' | 'error'

export function ImportExportPage() {
  const [importStatus, setImportStatus] = useState<ImportStatus>('idle')
  const [importError, setImportError] = useState('')

  async function handleExport() {
    const bundle = await buildExportBundle()
    const json = JSON.stringify(bundle, null, 2)
    const blob = new Blob([json], { type: 'application/json' })
    const url = URL.createObjectURL(blob)
    const link = document.createElement('a')
    link.href = url
    link.download = `workday-autofill-backup-${bundle.exportedAt.slice(0, 10)}.json`
    link.click()
    URL.revokeObjectURL(url)
  }

  async function handleImportFile(event: ChangeEvent<HTMLInputElement>) {
    const file = event.target.files?.[0]
    if (!file) return

    try {
      const text = await file.text()
      const parsed: unknown = JSON.parse(text)
      const result = exportBundleSchema.safeParse(parsed)
      if (!result.success) {
        setImportStatus('error')
        setImportError('This file is not a valid backup.')
        return
      }
      await restoreFromBundle(result.data as unknown as ExportBundle)
      setImportStatus('success')
      setImportError('')
    } catch {
      setImportStatus('error')
      setImportError('This file is not a valid backup.')
    } finally {
      event.target.value = ''
    }
  }

  return (
    <section className="bg-raised border border-[#E3DFD8] rounded-panel p-6 flex flex-col gap-4 max-w-xl">
      <div className="flex flex-col gap-1">
        <h2 className="m-0 text-[20px] font-semibold tracking-[-0.02em]">Import / Export</h2>
        <span className="text-[13px] text-muted">Your data lives in this browser. Take it with you.</span>
      </div>

      <div className="bg-surface border border-line rounded-card p-4 flex items-center justify-between gap-3">
        <span className="text-[13px] font-semibold">Export your data</span>
        <button
          type="button"
          onClick={handleExport}
          className="font-sans text-[13px] font-semibold bg-ink text-white rounded-input px-4 py-2.5 hover:bg-[#2E2B26] transition-colors duration-150 flex-shrink-0"

        </button>
      </div>

import type {
  ApplicationSavedMessage,
  AutofillResultMessage,
  PageStatusMessage,
} from '../shared/messaging/messages'

type Status = 'loading' | 'workday-detected' | 'not-workday'

  'font-sans text-[14px] font-semibold bg-ink text-white rounded-xl px-3.5 py-3.5 hover:bg-[#2E2B26] transition-colors duration-150 w-full'

  return (

      <span className={`text-[20px] font-bold tracking-[-0.02em] ${color}`}>{value}</span>
      <span className="font-mono text-[10px] text-muted">{label}</span>

export function App() {
  const [status, setStatus] = useState<Status>('loading')
  const [tabId, setTabId] = useState<number | undefined>(undefined)
  const [summary, setSummary] = useState<AutofillResultMessage['summary'] | undefined>(undefined)
  const [savedRecord, setSavedRecord] = useState<ApplicationSavedMessage['record'] | undefined>(
    undefined
  )

  useEffect(() => {
    let cancelled = false

    async function checkPageStatus() {
      const [tab] = await chrome.tabs.query({ active: true, currentWindow: true })
      if (!tab?.id) {
        if (!cancelled) setStatus('not-workday')
        return
      }
      if (!cancelled) setTabId(tab.id)
      try {
        const response = (await chrome.tabs.sendMessage(tab.id, {
          type: 'GET_PAGE_STATUS',
        })) as PageStatusMessage | undefined
        if (!cancelled) {
          setStatus(response?.isWorkdayPage ? 'workday-detected' : 'not-workday')
        }
      } catch {
        // No content script is listening on this tab — it only injects on
        // *.myworkdayjobs.com, so any other page rejects here rather than
        // resolving. Treat that the same as "not a Workday page".
        if (!cancelled) setStatus('not-workday')
      }
    }

    checkPageStatus()
    return () => {
      cancelled = true
    }
  }, [])

  async function handleAutofill() {
    if (!tabId) return
    try {
      const response = (await chrome.tabs.sendMessage(tabId, {
        type: 'AUTOFILL_PAGE',
      })) as AutofillResultMessage | undefined
      if (response?.summary) {
        setSummary(response.summary)
      }
    } catch {
      // The tab may have navigated away or the content script may no longer
      // be listening — nothing to update; the button stays clickable so the
      // user can retry.
    }
  }

  async function handleSaveApplication() {
    if (!tabId) return
    try {
      const response = (await chrome.tabs.sendMessage(tabId, {
        type: 'SAVE_APPLICATION',
      })) as ApplicationSavedMessage | undefined
      if (response?.record) {
        setSavedRecord(response.record)
      }
    } catch {
      // Same as handleAutofill: the tab may have navigated away or the
      // content script may no longer be listening.
    }
  }

  return (
    <div className="flex flex-col min-h-screen">
      <div className="flex items-center justify-between px-4.5 py-3.5 border-b border-line bg-surface">
        <Wordmark />
        <span className="font-mono text-[10px] text-muted">v1.0.0</span>
      </div>

      <div className="p-4.5 flex flex-col gap-3.5">
        {status === 'loading' && (
          <div className="flex items-center gap-2.5">
            <span className="w-2 h-2 rounded-full bg-[#C9C3BA]" />
            <p className="font-mono text-[12px] text-muted m-0">Checking current page…</p>
          </div>
        )}

        {status === 'not-workday' && (
          <>
            <div className={cardClass}>
              <span className="inline-flex items-center gap-2 text-[13px] font-semibold">
                <span className="w-2 h-2 rounded-full bg-[#C9C3BA]" />
                No Workday page detected.
              </span>
            </div>
            <button
              type="button"
              onClick={() => chrome.runtime.openOptionsPage()}
              className={secondaryButtonClass}
            >
              Open profile
            </button>
          </>
        )}

        {status === 'workday-detected' && (
          <>
            <div className={cardClass}>
              <span className="inline-flex items-center gap-2 text-[13px] font-semibold">
                <span className="w-2 h-2 rounded-full bg-success" />
                Workday page detected.
              </span>
            </div>
            <button type="button" onClick={handleAutofill} className={primaryButtonClass}>
              Autofill current page
            </button>
            {summary && (
              <div className={cardClass}>
                <span className="font-mono text-[11px] text-muted uppercase tracking-[0.08em]">
                  Last run
                </span>

                  <StatBox value={summary.filled} label="filled" color="text-success" />

                  <StatBox value={summary.skipped} label="skipped" color="text-faint" />
                </div>
                {summary.skipped > 0 && (
                  <p className="text-[12px] text-muted m-0">Some fields were skipped.</p>
                )}
                {summary.hasMoreEntries && (
                  <div className="border-t border-hairline pt-2.5 text-[12px] text-muted leading-normal">
                    If Workday has additional entries to fill, click &quot;Add&quot; on the page for
                    the next Work Experience or Education entry, then click Autofill again.
                  </div>
                )}
              </div>
            )}
            <button type="button" onClick={handleSaveApplication} className={secondaryButtonClass}>
              Save application
            </button>
            {savedRecord && (
              <>
                <div className="bg-[#F0F7F4] border border-[#CFE5DB] rounded-card px-3.5 py-3 flex items-center gap-2.5">
                  <span className="w-2 h-2 rounded-full bg-success flex-shrink-0" />

                >
                  Open profile →
                </button>
              </>
            )}
          </>
        )}
      </div>
    </div>
  )
}
